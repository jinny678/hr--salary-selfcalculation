# 薪酬计算交叉验证 Skill

## 概述

本 Skill 用于广汽五羊-本田正式工月度薪酬核验，自动读取月度工资核对工作簿（`.xls`/`.xlsx`），执行以下验证并生成报告：

1. **PS系统内部一致性验证** — 验证各分项之和是否等于总项
2. **PS与手工核对差异提取** — 提取Excel中"核对"列的差异值
3. **个税验算** — 按累计预扣法独立验算个税
4. **环比异常检测** — 与上月对比，标记变动超过阈值的异常项
5. **请款汇总** — 按员工个人和部门分别汇总应发/扣款/实发等，可直接用于ERP录入
6. **特殊状态员工标记** — 标记全月病假/产假/残疾人员工

## 触发条件

当用户执行以下操作时，调用本 Skill：

- 上传月度工资核对工作簿并要求核验/验证
- 要求计算个税、检测差异、生成请款汇总
- 询问薪酬数据是否有异常或需要交叉验证
- 要求执行月度薪酬自动化审核流程

## 使用流程

### 1. 接收文件

用户上传月度工资核对工作簿（`.xls` 或 `.xlsx`），将其保存到 `input/` 目录。

### 2. 确认配置

检查 `config.yaml` 中的关键参数，特别是 `tax_months_ytd`（累计月数）是否与当前月份一致。如用户未指定月份，使用当前月份。

### 3. 运行主程序

```bash
cd .trae/skills/salary-calculation
python main.py
```

### 4. 解读报告

运行完成后，读取生成的报告文件（`output/薪酬验证报告_YYYYMM.xlsx`），向用户汇报关键发现：

- 总览仪表盘中的 KPI 指标
- 需要关注的差异项（PS核对差异、个税差异、环比异常）
- 特殊状态员工情况
- 请款汇总金额

## 验证逻辑说明

### PS内部一致性
已验证 PS 系统内部计算 100% 正确（分项之和 = 总项，0差异）。

### 应税收入验证
```
应税收入 = 应发合计2 + 并入纳税
```
不一致记录标记为需核实。

### 个税验算（累计预扣法）
```
累计应纳税所得额 = 累计应税收入 - 起征点×月数 - 累计三险一金 - 累计专项扣除
累计应纳税额 = 累计应纳税所得额 × 税率 - 速算扣除数
本月个税 = 累计应纳税额 - 已缴纳税额
```
差异 ≤ 0.01 元为正常四舍五入，可忽略。

### 环比异常
变动率 > 10% 且变动金额 > 阈值的项目标记为异常。当前检测项目：基本工资、绩效工资、应发工资、实发金额、个税、扣缺勤、全勤津贴、并入纳税。

### 特殊状态
全月病假、产假、残疾人用工三类员工单独标记，已知 PS 系统对这类员工的计算偶有问题，需人工复核。

## 配置文件关键参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `tax_months_ytd` | 累计月数 | 6 |
| `tax_free_threshold` | 个税起征点/月 | 5000 |
| `anomaly_change_pct` | 环比异常变动率阈值(%) | 10 |

## 注意事项

1. 所有数据处理均在本地完成，不上传外部服务器
2. 报告为辅助验证，仍需人工审核确认
3. 待岗人员、非正式工不在覆盖范围内
4. 每月运行前需更新 `config.yaml` 中的 `tax_months_ytd`
5. 如果 Excel 报表格式变化，需调整 `config.yaml` 中的 `column_mapping` 列索引
