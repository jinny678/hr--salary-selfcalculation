# 薪酬计算交叉验证工具 — 使用说明

## 一、工具概述

本工具用于广汽五羊-本田正式工月度薪酬核验，自动读取月度工资核对工作簿，执行以下验证：

1. **PS系统内部一致性验证** — 验证各分项之和是否等于总项
2. **PS与手工核对差异提取** — 提取Excel中"核对"列的差异值
3. **个税验算** — 按累计预扣法独立验算个税
4. **环比异常检测** — 与上月对比，标记变动超过阈值的异常项
5. **请款汇总** — 按部门汇总应发/扣款/实发等，可直接用于ERP录入
6. **特殊状态员工标记** — 标记全月病假/产假/残疾人员工

---

## 二、环境要求

- **Python 3.8 或更高版本**
- 依赖包：pandas、openpyxl、pyyaml（首次运行自动安装）

---

## 三、快速开始

### 方式一：双击运行（推荐）

**Windows**：双击 `run.bat`

**Mac/Linux**：终端中执行 `./run.sh`（首次需执行 `chmod +x run.sh`）

### 方式二：命令行运行

```bash
# 1. 安装依赖（首次运行需要）
pip install -r requirements.txt

# 2. 运行
python main.py
```

---

## 四、使用步骤

### 第1步：放入输入文件

将月度工资核对工作簿（`.xls` 或 `.xlsx`）放入 `input` 文件夹。

> 文件名无要求，工具会自动识别。如果 `input` 文件夹中有多个文件，工具会使用第一个。

### 第2步：检查配置（按需）

打开 `config.yaml`，确认以下参数：

| 参数 | 说明 | 当前值 |
|------|------|--------|
| `tax_months_ytd` | 累计月数（月份） | 6 |
| `tax_free_threshold` | 个税起征点 | 5000元 |
| `anomaly_change_pct` | 环比异常变动率阈值 | 10% |

**每月需更新 `tax_months_ytd`**：1月填1，2月填2，依此类推。

### 第3步：运行工具

双击 `run.bat`（Windows）或执行 `./run.sh`（Mac/Linux）。

### 第4步：查看报告

运行完成后，在 `output` 文件夹中找到生成的报告文件（如 `薪酬验证报告_202606.xlsx`）。

报告包含以下工作表：

| 工作表 | 内容 |
|--------|------|
| 总览仪表盘 | 关键指标概览 + 验证结论 |
| PS核对差异 | PS系统值与手工核对值的差异明细 |
| 个税验算 | 累计预扣法验算个税差异 |
| 应税收入不一致 | 应税收入 ≠ 应发合计2+并入纳税 的记录 |
| 环比异常 | 与上月对比异常变动 |
| 请款汇总-个人 | 按员工ID逐人汇总的请款明细 |
| 请款汇总-部门 | 按部门汇总的请款数据 |
| 特殊状态员工 | 全月病假/产假/残疾人薪资明细 |
| 工具说明 | 各验证项的方法说明 |

### 第5步：历史存档

报告同时自动保存到 `history` 文件夹，方便后续追溯对比。

## 五、配置文件说明

所有规则参数均在 `config.yaml` 中配置，无需修改代码：

| 配置项 | 说明 |
|--------|------|
| `tax_brackets` | 个税税率表（累计预扣法），按国家政策调整 |
| `tax_months_ytd` | 累计月数，每月更新 |
| `tax_free_threshold` | 个税起征点/月 |
| `anomaly_items` | 环比异常检测项目及阈值 |
| `rules` | 给予规则参数（加班费率、津贴标准等） |
| `column_mapping` | Excel列索引映射（如报表格式变化需调整） |
| `check_items` | 需要检测核对差异的项目列表 |
| `summary_columns` | 请款汇总的列 |

## 六、常见问题

**Q: 运行报错"未找到 .xls/.xlsx 文件"**
A: 请确保已将工资核对工作簿放入 `input` 文件夹。

**Q: 个税差异很大怎么办？**
A: 先检查 `config.yaml` 中 `tax_months_ytd` 是否为当前累计月数。如果确认无误，差异>0.5元的需要人工复核。

**Q: 环比异常很多怎么办？**
A: 可调整 `config.yaml` 中的 `anomaly_items` 阈值，或提高 `anomaly_change_pct` 来减少误报。

**Q: Excel报表格式变了怎么办？**
A: 修改 `config.yaml` 中的 `column_mapping` 列索引，使其与新的报表格式一致。

**Q: 工具支持哪些Excel格式？**
A: 支持 `.xls` 和 `.xlsx` 两种格式。

---

## 七、注意事项

1. 所有数据处理均在本地完成，不会上传到任何外部服务器
2. 工具生成的是辅助验证报告，仍需人工审核确认
3. 特殊情况（如待岗人员、非正式工）的薪资计算不在此工具覆盖范围内
4. 个税差异 <= 0.01 元属于正常四舍五入，可忽略
5. 每月运行前请更新 `config.yaml` 中的 `tax_months_ytd`
