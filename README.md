# 薪酬计算交叉验证工具


## 功能

| 验证项 | 方法 | 说明 |
|--------|------|------|
| PS内部一致性 | 分项之和 = 总项 | 已验证通过（0差异） |
| 应税收入验证 | 应发合计2 + 并入纳税 | 标记不一致记录 |
| PS与手工核对 | 提取Excel中核对列差异 | 反映人工修正项 |
| 个税验算 | 累计预扣法 | 差异 > 0.1元标记 |
| 环比异常检测 | 变动率 > 10%且金额 > 阈值 | 需人工确认 |
| 请款汇总 | 按部门汇总 | 可直接用于ERP |

## 快速开始

### 方式一：TRAE Skill（推荐）

在 TRAE 中上传月度工资核对工作簿，Skill 会自动触发并执行验证。

### 方式二：独立运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 将工资核对工作簿放入 input/ 文件夹

# 3. 运行
python main.py
# 或双击 run.bat（Windows）/ 执行 ./run.sh（Mac/Linux）
```

## 报告内容

运行后生成 `output/薪酬验证报告_YYYYMM.xlsx`，包含 9 个工作表：

- **总览仪表盘** — KPI 概览 + 验证结论
- **PS核对差异** — 系统值与手工核对值差异明细
- **个税验算** — 累计预扣法验算个税差异
- **应税收入不一致** — 应税收入 ≠ 应发合计2+并入纳税 的记录
- **环比异常** — 与上月对比异常变动
- **请款汇总-个人** — 按员工ID逐人汇总的请款明细
- **请款汇总-部门** — 按部门汇总的请款数据
- **特殊状态员工** — 全月病假/产假/残疾人薪资明细
- **工具说明** — 各验证项的方法说明

## 配置

所有规则参数在 `config.yaml` 中配置，无需修改代码。每月运行前需更新 `tax_months_ytd`（累计月数）。

## 注意事项

- 所有数据处理在本地完成，不上传外部服务器
- 报告为辅助验证，仍需人工审核
- 待岗人员、非正式工不在覆盖范围内
- 个税差异 ≤ 0.01 元为正常四舍五入，可忽略

## 环境要求

- Python 3.8+
- pandas >= 1.5.0
- openpyxl >= 3.0.0
- pyyaml >= 6.0
